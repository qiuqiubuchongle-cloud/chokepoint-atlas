# Catalyst Watch

Use this file when the user wants follow-up monitoring, a watchlist, or a continuing lane tracker.

## Goal

Move from one-off analysis to timeline-based monitoring.

## Catalyst Types

- earnings date
- qualification milestone
- capacity-expansion announcement
- product launch
- customer onboarding
- first volume shipment
- policy / funding event
- supplier-list change

## What To Track

For each lane, maintain:

1. the next three catalyst events
2. what confirmation would strengthen the thesis
3. what negative surprise would weaken it
4. which companies should react first

## Output Pattern

```text
Catalyst Watch

Next events:
- Q2 earnings: watch for lead-time language
- customer launch: watch for supplier mentions
- capacity update: watch for allocation or booked-out wording

Bullish confirmation:
- management confirms qualification completed

Bearish confirmation:
- customer signals dual-sourcing or no bottleneck

Most sensitive names:
- ...
```
